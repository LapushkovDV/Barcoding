Обоснование требований

1. Atlantis не ниже 5.5.29.3. + обязательно обновить napsrv.exe не ниже 5.5.29.0.
	ПИР 102.176812 - В версиях Atlantis до 5.5.29.3 не реализована поддержка работы Атлантиса в произвольном потоке через контекст.
	ПИР 102.170169 - В версиях Atlantis до 5.5.29.0 генерируется SQL, который приводит к падению адаптера на Oracle. 
	ПИР 102.168417 - При napsrv версии ниже 5.5.29.0 Ошибка записи в базу данных из-за использования занятых адресов.
	ПИР 102.174952 - При napsrv версии ниже 5.5.29.0 при вставке записи во временную таблицу возникает исключение (DyplicateKey).

2. Для MS SQL необходимо использовать Microsoft SQL Server Native Client Version >=11.00.6540
	ПИР 102.111785 - В многопоточном режиме стандартный драйвер работает некорректно.

3.	Файл GalaktikaApi.Generated.cs сгенерирован на основании Galaktika 9.2. Версия AlterCumulative 9.2.3
	Полный отчет расположен вместе с адаптером ERP.
	Если версия AlterCumulative не соответствует той, на основании которой сгенерирован файл, то это может привести к некорректному 
	выполению DSQL-запросов. Для перегенерации воспользоваться инструкцией "Построить адаптер при помощи сборщика". Инструкция 
	расположена в папке Doc сборки ESB.
